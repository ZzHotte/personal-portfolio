Shared Whiteboard

Distributed application · System design

Overview

A real-time collaborative whiteboard where multiple clients draw and see updates with low latency. The design separates transport, session state, and persistence so the system can scale horizontally and recover cleanly from partial failures.

Architecture

The first diagram focuses on the distributed platform: service registration (bind), discovery (lookup), and server-to-client callback/RPC for live updates. The second diagram shows how editors and viewers interact with the app over WebSockets and HTTPS, and how the client connects to storage and backend services.

Challenges

  • Ordering and merging concurrent edits without losing user intent.
  • Connection churn, reconnects, and idempotent handling of client events.
  • Back-pressure and fan-out when many users share the same board.
